## Description


We want to explore the importance of shifting security left in DevOps pipelines, focusing on how container vulnerability scanning can prevent critical security incidents.


We will explain why traditional DevOps pipelines often ignore security, how vulnerabilities in base images (such as Log4j or outdated Node.js versions) can slip into production, and how automated scanning addresses this challenge. As a concrete example, we will show how Trivy can be integrated into CI/CD to automatically detect vulnerabilities before deployment.


**Relevance**


The rise of DevSecOps shows that security must be part of the pipeline, not an afterthought. Container vulnerability scanning directly addresses this need by ensuring that the artifacts delivered through continuous deployment are safe and reliable. By combining Trivy as a case study, our presentation highlights how teams can combine speed and security, preventing future incidents while maintaining delivery velocity.
