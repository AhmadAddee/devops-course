## Description

The presentation introduces Kusari as a tool to secure the dependencies in a project, ensuring that vulnerabilities are found and tracked. We emphasise how Kusari uses SBOMs to scan open-source dependencies, finding the most critical vulnerabilities and correlating these to show which attacks are most probable. The main focus of the presentation is on how does Kusari work and which methodology it uses, as well as the limitations that may arise when using this tool.

**Relevance**

Using tools like Kusari is highly relevant in DevSecOps because it provides the developers with information about the security risks associated with integrating third party softwares. It provides visibility correlating which vulnerabilities are there, where they come from, and how they relate to each other.
