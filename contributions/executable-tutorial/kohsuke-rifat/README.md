## Description
In this tutorial, you'll learn how to use LicenseFinder to automatically detect licenses for your project's dependencies and set policies based on the actual licenses.

When you work at a company, you encounter software licensing issues. As applications grow in scale, dependency management becomes increasingly complex. Even if your own code is secure, ignoring the licensing terms of libraries can lead to legal risks and compliance issues. Through this tutorial, participants will learn how to automate dependency license management and reduce risks.

Tutorial Content:

-   Dependency License and Legal Risk Explanation
-   Introduce LicenseFinder and set up project
-   Scanning Dependencies and Generating Reports Using LicenseFinder
-   Configuring Permitted/Prohibited License Policies from real licence
-   For more complex dependency licenses problem

The tutorial is delivered on the KillerKoda platform.

**Relevance**

In DevOps, the focus is typically on ensuring software functions properly, while license issues for enterprise software are often overlooked by engineers. Integrating dependency license checks into DevOps can eliminate the risk of lawsuits for companies.
