## Description

It's now becoming easier to integrate LLM capability into a product's offering - but is it easy to keep them secure?

With the rise of LLM integrations, it's also important to safeguard the interactions between the user and the system so malicious attacks through LLMs can be prevented. One recent case study revealed a successful takeover of an AI sales agent to reveal its own private customer data through prompt injection [(source)](https://x.com/mbrg0/status/1953815729947447770).

Related to this, we'd like to demo [Superagent](https://www.superagent.sh) which introduces itself as a _proxy_ between a service and the LLM provider to secure both the service owner and the end user. It offers a practical solution to add security protection to prevent prompt injection and personal data leaks through LLM. We want to see whether Superagent is indeed easy to setup and beneficial as they are advertised.

**Relevance**

We think that DevSecOps could be done at any stage of development and is a _continous concern_. Tools like Superagent can help dev team members to secure their system and users in any stage where LLMs are involved in the process.
